# Metodología

## Objetivo
- Comparar modelos en tareas prácticas (código, datos, comunicación, asistencia).
- Mantener trazabilidad: configuración, prompts, outputs, métricas, costo y latencia.

## Principios
- Reproducibilidad: cada run guarda commit, modelo y parámetros.
- Evaluación automatizada cuando sea posible.
- Rúbricas humanas cuando el juicio sea inevitable (tono, claridad, estrategia).

## Niveles de costo
- Smoke (barato): pocas tareas, pocos casos.
- Full (caro): todo el catálogo.
